## 🐛 发现的问题与修复

### 问题 1: Windows 编码兼容性问题
**现象**: 脚本在 Windows PowerShell 中运行时，中文和 emoji 字符输出乱码

**原因**: Python 默认使用 GBK 编码输出，但脚本包含 UTF-8 字符（emoji、中文）

**修复方案**: 在所有脚本的 `__main__` 块中添加：
```python
import sys
sys.stdout.reconfigure(encoding='utf-8')
```

**修复文件**:
- `scripts/risk_scorer.py` ✅
- `scripts/position_calculator.py` ✅
- `scripts/stoploss_calculator.py` ✅
