## 🎯 系统定位

### 核心理念
> **不做选股专家，做配置专家**
> 
> 从"这只股票好不好"转向"这个组合健不健康"

### 与 TradingAgents 的差异化

| 维度 | TradingAgents | 本系统 |
|------|---------------|--------|
| **焦点** | 个股分析 | 组合配置 |
| **目标** | 交易决策 | 资产配置 |
| **方法** | 多智能体辩论 | 资产配置理论 |
| **输出** | 买入/卖出/持有 | 配置比例/权重/再平衡 |
| **时间** | 短期交易 | 长期投资 |

## 🤖 多智能体设计

### 智能体角色与职责

#### 1. 市场分析智能体
**职责**: 宏观环境判断
- 经济周期识别（美林时钟）
- 市场周期判断（牛/熊/震荡）
- 货币政策分析
- 估值水平评估

**输出**: 《市场环境分析报告》
- 当前周期阶段
- 市场情绪指标
- 建议股票仓位区间

#### 2. 行业分析智能体
**职责**: 行业轮动与板块分析
- 行业景气度分析
- 行业轮动判断
- 板块估值比较
- 政策影响评估

**输出**: 《行业配置建议》
- 超配行业推荐
- 低配行业建议
- 行业权重范围

#### 3. 组合经理智能体
**职责**: 资产配置决策
- 战略资产配置 (SAA)
- 战术资产配置 (TAA)
- 组合权重优化
- 再平衡触发判断

**输出**: 《资产配置方案》
- 股/债/现比例
- 行业配置权重
- 建仓/调仓建议

#### 4. 风险经理智能体
**职责**: 风险评估与监控
- VaR 计算
- 最大回撤预估
- 波动率分析
- 压力测试
- 相关性监控

**输出**: 《风险评估报告》
- 组合风险指标
- 风险预警信号
- 风险对冲建议

#### 5. 再平衡智能体
**职责**: 动态调整监控
- 定期再平衡提醒
- 阈值再平衡触发
- 税收优化建议
- 交易成本估算

**输出**: 《再平衡建议》
- 需要调整的仓位
- 具体操作建议
- 成本与收益分析

## 📊 核心功能模块

### 1. 投资者画像系统
```python
class InvestorProfile:
    - 风险承受能力 (保守/稳健/平衡/成长/激进)
    - 投资目标 (保值/增值/高增长)
    - 投资期限 (短期/中期/长期)
    - 流动性需求 (高/中/低)
    - 投资偏好 (行业/风格/地区)
    - 投资禁忌 (避免的行业/策略)
```

### 2. 战略资产配置 (SAA)
```python
class StrategicAssetAllocation:
    - 基于投资者画像的长期配置
    - 参考标准配置比例
    - 考虑历史回报和相关性
    - 定期回顾（年度）
```

**标准配置参考**:
| 类型 | 股票 | 债券 | 现金 | 其他 |
|------|------|------|------|------|
| 保守型 | 20% | 60% | 20% | 0% |
| 稳健型 | 50% | 40% | 10% | 0% |
| 平衡型 | 60% | 30% | 10% | 0% |
| 成长型 | 75% | 20% | 5% | 0% |
| 激进型 | 90% | 5% | 5% | 0% |

### 3. 战术资产配置 (TAA)
```python
class TacticalAssetAllocation:
    - 基于市场环境的动态调整
    - 牛市：提高股票仓位
    - 熊市：提高债券/现金仓位
    - 震荡市：均衡配置
    - 调整幅度：±10-20%
```

### 4. 组合优化引擎
```python
class PortfolioOptimizer:
    - 马科维茨均值 - 方差优化
    - Black-Litterman 模型
    - 风险平价模型
    - 约束条件:
      * 单一行业 ≤ 30%
      * 单只股票 ≤ 20%
      * 最小持仓数量
```

### 5. 风险管理系统
```python
class RiskManagement:
    - VaR (Value at Risk)
    - ES (Expected Shortfall)
    - 最大回撤 (Max Drawdown)
    - 波动率 (Volatility)
    - 夏普比率 (Sharpe Ratio)
    - 贝塔系数 (Beta)
    - 相关性矩阵
```

### 6. 再平衡系统
```python
class Rebalancing:
    - 定期再平衡 (季度/年度)
    - 阈值再平衡 (偏离>5%)
    - 动态再平衡 (根据估值)
    - 税收优化策略
    - 交易成本控制
```

## 🔄 核心工作流程

### 流程 1: 新组合创建
```
1. 用户填写风险问卷
   ↓
2. 生成投资者画像
   ↓
3. 确定战略资产配置 (SAA)
   ↓
4. 分析当前市场环境
   ↓
5. 制定战术资产配置 (TAA)
   ↓
6. 优化组合权重
   ↓
7. 生成建仓计划
   ↓
8. 输出完整配置方案
```

### 流程 2: 组合监控
```
1. 每日获取市场数据
   ↓
2. 计算组合现值
   ↓
3. 评估风险指标
   ↓
4. 检查再平衡条件
   ↓
5. 生成监控报告
   ↓
6. 触发预警 (如需要)
```

### 流程 3: 组合调整
```
1. 检测再平衡触发条件
   ↓
2. 分析偏离原因
   ↓
3. 计算目标权重
   ↓
4. 生成调仓建议
   ↓
5. 估算交易成本
   ↓
6. 输出调整方案
   ↓
7. 等待用户确认
```

### 流程 4: 定期复盘
```
1. 收集周期数据
   ↓
2. 计算周期收益
   ↓
3. 归因分析 (配置/选股/时机)
   ↓
4. 风险评估
   ↓
5. 对比基准
   ↓
6. 总结经验教训
   ↓
7. 调整策略 (如需要)
```

## 🛠️ 技术栈选择

### 后端
- **语言**: Python 3.10+
- **框架**: FastAPI (可选，如需 Web API)
- **数据处理**: pandas, numpy
- **优化算法**: scipy.optimize, cvxpy

### 数据存储
- **缓存**: 文件系统 (默认) / Redis (可选)
- **数据库**: MongoDB (可选) / SQLite (可选)
- **配置**: YAML / JSON

### 数据源
- **A 股**: Tushare / AKShare / BaoStock

### 前端 (可选)
- **Web**: Vue 3 + Element Plus
- **CLI**: Typer / Click
- **API**: RESTful / GraphQL

## 🔒 风险控制

### 系统风险
- 数据源失败 → 多源降级
- 计算错误 → 双重验证
- 配置丢失 → 定期备份

### 投资风险
- 明确系统定位：辅助决策，不代客理财
- 充分风险提示：所有建议仅供参考
- 设置投资限额：大额操作需二次确认
